Getting oriented

1What SpookiUI is

SpookiUI is a single-file terminal app that turns Ghostty's plain-text config into a guided, safe, live-updating experience.

Ghostty is configured through ~/.config/ghostty/config and can't watch that file for changes — you normally edit text and trigger a reload by hand. SpookiUI closes that loop: it writes the file and triggers the reload for you, so when you run it inside a Ghostty window the very terminal you're in repaints as you edit.

Every option, discovered live

Options aren't hard-coded. SpookiUI reads them from your installed Ghostty (ghostty +show-config --default --docs), so it always matches your version — around 200 options across 13 categories, each with its docs.

Safe by construction

Every change is validated by Ghostty itself before it's saved. An invalid value is rejected and rolled back — it never reaches your config. A dated backup is kept, and you can revert a whole session or restore Ghostty's defaults at any time.

Two ways to drive it. The interactive TUI (just run spookiui) and a scriptable CLI (spookiui set, get, doctor, …) share the exact same core logic, so anything you can click you can also script.

Requirements

  • Python 3.8+ — standard library only, no pip install ever.
  • The ghostty binary on your PATH (or /Applications/Ghostty.app).
  • macOS or Linux. Live reload works on both (details in §7).
Setup

2Installing

Run straight from the repo

bash
$ git clone https://github.com/mattj85/SpookiUI.git
$ cd SpookiUI
$ ./spookiui.py

Put spookiui on your PATH

bash
$ ./install.sh                     # installs to ~/.local/bin
$ PREFIX=/usr/local ./install.sh   # system-wide (may need sudo)

The installer checks Python 3.8+, warns if ghostty is missing, and symlinks the script onto your PATH. To reverse it, run ./uninstall.sh (it only removes the symlink it created — never your config, and never a Homebrew install).

Homebrew (macOS & Linux)

bash
$ brew install mattj85/spookiui/spookiui
$ brew upgrade  mattj85/spookiui/spookiui   # later, to update
Installed via Homebrew? Update with brew upgrade — SpookiUI's in-app updater detects a Homebrew install and defers to it, so the two never fight.

Making changes

4Editing options

Press Enter on any option and SpookiUI opens the right editor for its type. When auto-apply is on (the default), the change is validated, written, and reloaded live as you go; pickers even preview the highlighted value and snap back if you cancel.

Staged edits. Press a to turn auto-apply off — edits are held in memory and nothing touches disk until you press s to save and reload them all at once. R reverts the whole session; X wipes back to Ghostty's defaults (a backup is always kept).
Managing setups

5Profiles

Profiles are named snapshots of your whole config. Save your current setup, tweak freely, and load any profile back later. Press p for the profiles view.

profiles
 profiles                                                                       
 saved in ~/.local/share/spookiui/profiles

 → dark            
   light
   work
 s save · Enter/l load · d delete · t light↔dark · Esc close

Save profiles named light and dark and the t key (or spookiui profile toggle) flips between them instantly — a one-key day/night switch. Loading a profile is validated and backs up your current config first, exactly like any other change. Profiles live outside Ghostty's config directory, so Ghostty never reads them.

Keeping it healthy

6The config doctor & utilities

Press c (or run spookiui doctor) for a health check. It reports invalid settings, unknown or misspelled options, options set more than once (dead lines), settings that merely repeat a default, and keybind triggers bound twice or shadowing a Ghostty default. It exits non-zero on errors, so it drops neatly into a dotfiles pre-commit hook.

spookiui doctor
 config check · 1 error(s), 3 warning(s)                                        
 ✗ invalid config: bogus-option: unknown field
 ! unknown option `bogus-option` — not recognised by your Ghostty
 ! `font-size` is set 2× — only the last takes effect; earlier ones are dead
 ! keybind trigger `super+k` is bound 2× — the later shadows the earlier
 · `cursor-style` is set to its default — redundant, can be removed

Utils

Press v for one-shot maintenance actions (such as Fix SSH) that solve common Ghostty papercuts without hunting through options.

Under the hood & scripting

7Live reload & the CLI

How the live reload fires

PlatformMechanismNeeds
macOSClicks Ghostty's Reload Configuration menu item via AppleScriptAccessibility permission for your terminal
LinuxSends SIGUSR2 to the running Ghostty processpgrep + a running Ghostty
OtherWrites & validates onlyTrigger your own reload keybind

If a reload can't fire, your change is still written and validated safely — SpookiUI just tells you to reload manually.

Scriptable CLI

Everything the TUI does is available non-interactively — handy for dotfiles and scripts.

CommandPurpose
spookiui list [category]List options (* = changed from default)

Staying current. On startup SpookiUI quietly checks GitHub for a newer release and shows an ↑ UPDATE badge if one exists. Update in place with U or spookiui update (or brew upgrade for Homebrew installs). Set SPOOKIUI_NO_UPDATE_CHECK=1 to disable the check.
